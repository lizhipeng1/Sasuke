使用介绍：
jar 用于扩展 消息通知
1.需要将redis的配置文件中的   notify-keyspace-events 设置为 "Ex"
2.在spring 项目中需要子类实现 ExpireKeyOperate 并实现方法
3.正常启动项目 key 过期的时候会 回调 ExpireKeyOperate 中的方法


对 redis 分布式部署 每个redis 进行 key expire  消息监听
对 失效的消息 做 分发



/////////////////////////////////////////////////////////
过期消息持久化
在服务挂了或者 重启服务的时候 会 丢失 redis  发布的  消息


bug问题 ：
1. redis取消订阅通知
2. tomcat关闭的时候要关闭 redis的订阅服务
3. 线程池统一定义 执行



需要一个统一管理的地方
1. 取消订阅的时候发送消息通知注册中心
2. 注册中心协调 其他的服务去 进行订阅服务操作


////////////////////////////////////////////////////
升级
获取所有的订阅者 存储记录
1. 当监听程序tomcat关闭的时候  清除记录
2. 选出下一个 监听程序 启动监听允许 处理业务逻辑
3. 持久化 失效的key  业务处理完成之后删除key
4. 未处理的key 触发业务处理
